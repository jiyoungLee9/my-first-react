나는 현재 JSP/jQuery 환경에 익숙한 웹퍼블리셔이고, React 기반의 웹퍼블리셔로 전환하기 위해 **'UI 컴포넌트 포트폴리오(총 12개)'**를 만드는 프로젝트를 진행 중이야.

이전 대화에서 진행하던 맥락을 이어받아, 다음 단계인 **Modal (Dialog)** 컴포넌트 제작을 가이드해 줘.

### 1. 프로젝트 환경 및 설정
* **개발 환경:** Windows 11, VS Code, Node.js, Vite 기반 React 프로젝트
* **스타일링:** SCSS Modules (`*.module.scss`) 사용
    * `vite.config.js` 설정을 통해 `variable.scss`의 변수($)와 믹스인(@mixin)을 모든 SCSS 파일에 자동 주입 중임.
* **문서화:** Storybook v8 사용.
    * 컴포넌트 구조: `[Component.jsx + Component.module.scss + Component.stories.jsx]` (1세트)
* **학습 선호:** 설명할 때 **"JSP/jQuery에서는 이랬는데, React에서는 이렇다"**는 비교 방식을 선호함.

### 3. ⭐ 필수 코딩 컨벤션 (Strict Rules)
우리는 다음 7가지 규칙을 철저히 지키며 코드를 작성하고 있어. 이 기준에 맞춰 가이드해 줘.

1.  **실무 지향:** 실제 개발자가 View단으로 바로 사용할 수 있도록, API/데이터 로직과 UI를 분리하여 설계.
2.  **Props 네이밍 (Boolean):** `is...` 접두사 사용 (예: `isOpen`, `isLoading`).
    * **단, HTML 표준 속성(`checked`, `disabled`, `required` 등)은 `is` 없이 그대로 사용.**
3.  **Props 네이밍 (Event):** `on...` 접두사 사용 (예: `onClick`, `onClose`).
4.  **확장성:** 개발자의 커스텀 스타일링을 위해 외부에서 들어오는 `className`을 내부 스타일과 병합 처리.
5.  **웹접근성(A11y):** `id`, `htmlFor`, `role`, `aria-*` 속성을 필수적으로 포함.
6.  **안전성:** Default Value가 없어도 UI가 깨지지 않도록 방어 코드 작성.
7.  **상태 관리:**
    * Form 요소(Input, Checkbox 등)는 철저한 **제어 컴포넌트(Controlled Component)**로 작성 (내부 state 지양).
    * UI 동작(Modal, Tabs)은 내부 state를 가지되, 외부 제어(`props`)도 가능하도록 유연하게 설계.
    * **Storybook:** `useState`를 `render` 함수 내부 템플릿에 사용하여 인터랙션이 실제로 동작하는지 보여줄 것.
